# Getting Started with Great Filter Navigator

Welcome. This guide is designed to get you productive in 15 minutes.

## ‚ö° 5-Minute Understanding

The **Great Filter Navigator (GFN)** is a framework for understanding and navigating existential risks to human civilization. We operate on a simple loop:

```
Hypothesize (10_) ‚Üí Monitor (20_) ‚Üí Intervene (30_) ‚Üí Observe (40_) ‚Üí Apply (50_)
```

**What this means:**
- **10_hypotheses**: "What could go wrong?" (12 failure modes)
- **20_mechanisms**: "How do we know danger?" (early-warning signals)
- **30_responses**: "How do we steer away?" (protocols & solutions)
- **40_analysis_logic**: "What's actually happening?" (empirical data)
- **50_application_guides**: "How do I use this?" (real-world guidance)

That's it. Everything fits into this loop.

1. Pick a hypothesis from [10_hypotheses/](./10_hypotheses/)
2. Read it carefully
3. Find a paper or source that supports or challenges it

3. Open an Issue with title: `[CLARIFICATION] Entry X1 uses term "[term]" ambiguously`
4. Suggest 2-3 possible interpretations

**This is valuable**: You've made the framework more rigorous.

## ‚ùì Common Questions

### "How do I know if I can edit directly vs. opening an Issue?"

**Open an Issue first for:**
- New hypotheses (requires high-stakes decision)
- Major changes to existing entries
- New categories or structural changes

**You can edit directly for:**
- Fixing typos and formatting
- Adding citations and references
- Clarifying existing explanations
- Linking entries together

**When in doubt**: Open an Issue. It's not extra work‚Äîit's how we maintain integrity.

---

### "What if I disagree with a hypothesis?"

**Do this:**
1. Open an Issue with `[CHALLENGE]` tag
2. State your counterargument clearly
3. Provide evidence or reasoning
4. Be respectful but direct

This is the exact kind of contribution we need. We want our hypotheses tested.

---

### "Can I add a new Great Filter hypothesis I discovered?"

**Yes, but follow the protocol:**
1. Open an Issue with `[NEW_HYPOTHESIS]` tag
2. State the hypothesis clearly
3. Explain the mechanism (how it could destroy civilization)
4. Provide evidence (papers, reasoning, examples)
5. Mark it `[Speculative]` if you're uncertain
6. Wait for discussion before submitting PR

The best new hypotheses build on evidence, not intuition alone.

---

### "How do I propose a monitoring indicator?"

**Protocol:**
1. Pick a hypothesis from 10_hypotheses/ that interests you
2. Open an Issue with `[NEW_INDICATOR]` tag
3. State what you want to measure
4. Explain data sources and feasibility
5. Describe the "tripwire" threshold (when this indicator means danger)
6. Link back to the hypothesis it monitors
7. Wait for feedback, then draft the entry

---

### "What's the difference between this and other AI alignment projects?"

**GFN is different because:**
- We assume traditional alignment mechanisms may fail (hard takeoff scenario)
- We're building a "shared map" rather than constraints
- We're trying to make survival **logically necessary** for any self-improving system
- We focus on observable, data-driven risk frameworks

See [README.md](./README.md) for the full context.

## üõ† Tools & Templates

### Before You Contribute

1. **Read** the relevant subsection README (10_/, 20_/, 30_/, etc.)
2. **Check** [INTERCONNECTION_MAP.md](./INTERCONNECTION_MAP.md) for status
3. **Use** `00_scope/entry_template.md` as your structure

1. Open an Issue (don't edit directly)
2. Tag it: `[NEW_HYPOTHESIS]`, `[NEW_INDICATOR]`, `[GAP]`, `[CHALLENGE]`, etc.
3. Be clear about what you're proposing and why
4. Link to related existing entries
5. Wait for discussion
